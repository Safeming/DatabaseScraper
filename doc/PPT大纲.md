**RPA 数据自动化采集工具的设计与实现**

---

## 第 2 页：项目背景与目标

- 传统数据采集痛点：手动复制粘贴、格式不统一、无法批量处理、重复劳动
- 项目目标：构建一个自动化采集工具，实现"输入 URL + 指定字段 → 自动输出结构化数据"
- 核心理念：让 AI 理解网页内容，按需提取，而非硬编码爬虫规则

---

## 第 3 页：系统架构总览

```
用户输入 URL + 字段
       ↓
┌─────────────────────────────┐
│  Streamlit Web UI           │
│  (快速抓取 / 批量采集 / 历史)│
└──────────────┬──────────────┘
               ↓
┌──────────────────────────────┐
│  Pipeline 流程编排引擎        │
│  抓取 → 清洗 → LLM提取 → 入库 │
└──────────────┬───────────────┘
               ↓
┌──────────────────────────────┐
│  数据持久层 (SQLite + Excel)  │
└──────────────────────────────┘
```

---

## 第 4 页：核心能力一 — 智能网页抓取

- 双引擎：Crawl4AI（AI 驱动）+ Selenium（浏览器模拟）
- 自动降级：Crawl4AI 超时/失败 → 自动切换 Selenium
- 自动翻页：检测"下一页"链接，支持最多 50 页连续采集
- 反爬应对：随机 User-Agent、Cookie 自动处理、滚动加载

---

## 第 5 页：核心能力二 — LLM 智能提取

- 不写 XPath/CSS 选择器，用自然语言描述需要的字段
- 支持本地模型（Ollama）和云端模型（Sambanova）
- 输入：清洗后的网页 Markdown + 用户指定字段
- 输出：标准 CSV 格式的结构化数据
- 优势：同一套工具适用于任意网站，无需针对每个站点开发

---

## 第 6 页：核心能力三 — 批量采集与调度

- 批量输入：支持多 URL 同时提交
- 定时调度：立即执行 / 每 N 小时 / 自定义 Cron 表达式
- 后台执行：提交即运行，无需等待
- 任务管理：实时查看进度、状态追踪、失败重试

---

## 第 7 页：核心能力四 — 流程编排（Pipeline）

五阶段流水线：

| 阶段 | 功能 | 容错机制 |
|------|------|---------|
| 1. 抓取 | 获取网页内容 | 双引擎降级、超时控制 |
| 2. 清洗 | 去噪、格式化 | 去除广告/导航/脚本 |
| 3. 提取 | LLM 结构化输出 | 格式校验、后处理修复 |
| 4. 入库 | SQLite 持久化 | 去重、增量追加 |
| 5. 导出 | Excel/CSV 输出 | 自动生成文件 |

---

## 第 8 页：核心能力五 — 数据入库与导出

- SQLite 零配置数据库，三表设计（任务/结果/数据）
- 自动去重：相同数据不重复入库
- 多格式导出：CSV、JSON、Excel、Markdown、HTML
- 任务历史：可追溯每次采集的原始内容和提取结果

---

## 第 9 页：技术栈

| 层次 | 技术选型 |
|------|---------|
| 前端 UI | Streamlit（Python Web 框架）|
| 网页抓取 | Crawl4AI + Selenium |
| 智能提取 | Ollama（本地）/ Sambanova（云端）|
| 数据存储 | SQLite + pandas |
| 任务调度 | schedule + croniter |
| 翻页检测 | BeautifulSoup + 正则匹配 |

## 第 11 页：实际效果展示

以 quotes.toscrape.com 为例：

- 输入：URL + 字段 "quote, author"
- 配置：自动翻页 7 页
- 输出：70 条结构化数据，quote 和 author 完美分离
- 耗时：约 2 分钟完成全部采集 + 提取

## 第 12 页：项目亮点总结

1. **智能化** — LLM 替代硬编码规则，一套工具适配任意网站
2. **自动化** — 批量提交、定时调度、自动翻页、后台执行
3. **可靠性** — 双引擎降级、错误隔离、空页终止、数据去重
4. **易用性** — Web UI 操作、零配置部署、多格式导出

---

## 第 13 页：不足与展望

- 当前局限：依赖 LLM 输出稳定性、不支持登录态网站、单机部署
- 未来方向：
  - 接入更多 LLM（GPT-4、Claude）
  - 支持需要登录的网站（Cookie 注入）
  - 分布式部署，提升并发能力
  - 可视化数据分析仪表盘
