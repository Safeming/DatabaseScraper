# AI-Scraper 项目修改记录

## 新增文件

- `sambanova.py` — SambaNova API 客户端封装（基于 OpenAI SDK）
- `.streamlit/config.toml` — Streamlit 配置，隐藏 Deploy 按钮和菜单
- `database.py` — SQLite 数据层，三表设计（jobs/results/extracted_data）
- `pipeline.py` — 五阶段流程编排引擎（抓取→清洗→提取→入库→导出）
- `pagination.py` — 翻页检测模块（HTML 解析 + URL 模式匹配）
- `scheduler.py` — 独立定时调度进程（轮询 + Cron 表达式）
- `start_scheduler.bat` — Windows 一键启动调度器
- `pages/1_批量采集.py` — 批量 URL 提交 + 翻页 + 调度配置
- `pages/2_任务历史.py` — 任务历史查看 + 数据预览 + 导出
- `categories.py` — 网站类别模板字典（books/news/jobs/products/forum 等 12 类），用于智能识别
- `tests/test_scraper_browser_setup.py` — Selenium/Crawl4AI 抓取链路回归测试
- `tests/__init__.py` — 单元测试包初始化文件

## Bug 修复

| 文件 | 问题 | 修复 |
|------|------|------|
| `scraper.py` | `raise('...')` 语法错误 | 改为 `raise ValueError(...)` |
| `app.py` | Ollama 路径硬编码，换机器就挂 | 用 `shutil.which()` + 常见路径 fallback |
| `app.py` | CSV 解析器无法处理 LLM 返回的 markdown 代码块和引号包裹 | 增加预处理逻辑 |
| `app.py` | `FileNotFoundError` 未捕获导致崩溃 | 异常类型补全 |
| `generate_response.py` | token 计数用了不支持的模型名 `gpt-4o-mini` | 改为 `gpt-4o` |
| `generate_response.py` | 模块加载时就初始化 SambaNova client，无 API Key 直接崩溃 | 改为懒加载 |
| `pipeline.py` | Crawl4AI 翻页只抓 1 页，`clean_markdown` 在翻页检测前就把链接格式破坏了 | 改为用 raw markdown 做翻页检测，清洗后的版本仅用于 LLM 提取 |
| `pipeline.py` | `find_next_page_in_markdown` 精确匹配 "next" 太严格 | 改为模糊匹配 `"next" in text_lower`，并增加 URL 模式递增 fallback |
| `pipeline.py` | `repair_csv_rows` 中含有 Unicode 弯引号导致 SyntaxError | 替换为 ASCII 引号和 `\x27` 转义 |
| `pipeline.py` | LLM 提取结果中 quote 列包含 "by Author Name" 重复归属 | 后处理自动检测并去除 |
| `scraper.py` | Windows 环境未安装 Chrome 时，Crawl4AI 失败后的 Selenium 降级仍会因找不到 Chrome 而失败 | 自动检测 Chrome/Edge，Chrome 不存在时使用本机 Microsoft Edge 启动 Selenium |
| `scraper.py` | Crawl4AI 在 Windows/GBK 控制台输出进度符号时可能触发编码异常 | 显式设置 `BrowserConfig(verbose=False)` 和 `CrawlerRunConfig(verbose=False)`，关闭进度符号输出 |
| `scraper.py` | Selenium 获取 HTML 失败后继续进入 `clean_html(None)`，导致错误不明确 | 空 HTML 或缺少 body 时抛出明确异常，并由 `selenium_scraper()` 返回 `Error:` 前缀错误 |

## 功能改进

| 改进项 | 说明 |
|--------|------|
| Prompt 优化 | 重写提取 Prompt，加入严格格式规则（列数一致、不合并字段、提取全部数据）|
| 超时控制 | Crawl4AI 120s 超时，Sambanova API 120s 超时，Ollama 加 `num_predict` 限制 |
| 输入校验 | URL 限制 2000 字符，query 限制 500 字符，空值检查 |
| 内存管理 | 消息超 40 条时自动保留最近 20 条，防止 session 膨胀 |
| UI 清理 | 隐藏 Deploy 按钮、三点菜单、顶部 header；去掉无意义的黄色/红色提示 |
| 日志规范 | `print()` 全部替换为 `logging`，prompt 只打印前 200 字符 |
| 错误处理 | Ollama 失败抛出明确异常，Sambanova 无 Key 时提前报错 |
| 代码清理 | 删除重复 if/else 分支，去掉无用注释 |
| 批量采集 | 多 URL 输入、后台线程执行、活跃任务状态实时显示 |
| 自动翻页 | 支持 Selenium（HTML 解析）和 Crawl4AI（Markdown 链接 + URL 模式）双路径 |
| 翻页终止 | 页面内容 < 100 字符自动停止；连续 2 页提取 0 行数据自动终止 |
| 数据去重 | `deduplicate_rows()` 对比数据库已有数据，JSON 序列化精确匹配去重 |
| CSV 后处理 | `repair_csv_rows()` 修复 LLM 输出问题：字段合并修复、转义符清理、空行过滤 |
| 定时调度 | 支持立即执行 / 每 N 小时 / 自定义 Cron，scheduler.py 独立进程轮询 |
| 数据导出 | 任务完成后自动导出 Excel；历史页支持 CSV/JSON/Excel 手动下载 |
| 任务管理 | 任务历史页：状态筛选、详情查看、数据预览、重新执行、删除 |
| Selenium 复用 | 新增 `SeleniumSession` 类，多页抓取时复用浏览器实例 |
| Crawl4AI 降级 | 抓取失败自动切换 Selenium，对用户透明 |
| Windows 浏览器兼容 | Selenium fallback 支持 Chrome 与 Microsoft Edge 自动选择，降低本机未安装 Chrome 时的失败率 |
| 抓取链路测试 | 新增单元测试覆盖 Edge fallback、Selenium 失败返回、Crawl4AI 静默配置 |

## 本次更新（智能识别 + LLM 校验重试 + 并发 + 登录态）

### 新增能力

| 能力 | 实现 | 涉及文件 |
|------|------|----------|
| 网站智能识别 | LLM 看页面前 2000 字符 → 输出类别 key（books/news/forum 等 12 类）→ 注入对应默认字段 | `categories.py`、`generate_response.py:classify_website()` |
| 字段自适应 | 分类后第二次问 LLM "实际页面有哪些字段"，仅在 general/forum 类或字段数 < 3 时触发；保守只加不减，最多新增 2 个 | `generate_response.py:adapt_fields_for_page()` |
| LLM 输出校验 | 检测列数对齐、空输出、表头与请求字段是否匹配、≥50% 行对齐 | `generate_response.py:validate_csv_output()` |
| LLM 重试机制 | 校验失败自动重试（默认 2 次），可配置 0-5 次 | `generate_response.py:generate_with_validation()` |
| 多 URL 并发 | `ThreadPoolExecutor` 同时处理多 URL（默认 3 路），URL 内部分页仍串行 | `pipeline.py:execute_pipeline()` |
| LLM 调用限流 | 全局信号量限制并发 LLM 调用数（默认 2），避免触发 rate limit | `pipeline.py:_LLM_SEMAPHORE` |
| SQLite 并发安全 | 全局锁保护写入，避免多线程下数据库竞态 | `pipeline.py:_DB_LOCK` |
| 登录态采集 | Cookie 字符串注入 Selenium，自动从 URL 推断 domain，提供 cookies 时强制走 Selenium | `scraper.py:SeleniumSession`、`pipeline.py:stage_scrape()` |

### Prompt 优化

| 优化点 | 说明 |
|--------|------|
| 反偷懒规则 | 显式禁止"只输出前几条"、禁止"把多字段塞到第一列" |
| 缺失字段处理 | 强制要求缺失字段填 `N/A`，不能跳过整行 |
| 完整性要求 | "页面有 30 条就输出 30 行，不允许提前停止" |
| 反幻觉 | 禁止编造、概括、改写，要求 verbatim 复制 |

  - Cookie 输入框 + 域名指定

  - LLM 重试次数（0-5，默认 2）

## 本次更新（数据库重构 — SQLite → MySQL 8.0）

为满足"数据库课程期末作品"的考核要求，把数据层从 SQLite 三表（业务数据塞 JSON）整体重构成 MySQL 8.0 关系型设计。

### 新增目录与文件

| 路径 | 说明 |
|------|------|
| `db/schema.sql` | 完整 DDL（15 表 + 4 视图 + 2 触发器 + 13 类别初始化） |
| `db/connection.py` | SQLAlchemy 引擎 / session 工厂，主连接 + 只读连接，从 `.env` 读配置 |
| `db/models.py` | 16 个 ORM 模型（一表一类） |
| `db/value_parsers.py` | 价格 / 评分 / 日期 / 量级后缀解析器，把 LLM 的字符串字段转 Python 类型 |
| `db/repository.py` | 兼容 `database.py` 的 15 个公开函数，内部走 SQLAlchemy |
| `db/stores/` | 类别专属入库（books / quotes / products / news / jobs_listings + generic 兜底） |
| `db/stores/__init__.py` | `STORE_REGISTRY` 路由表 + `store_by_category()` |
| `db/migration.py` | 老 SQLite 数据迁移脚本（按 query 推断 category 后路由到专属表） |
| `db/_legacy_sqlite.py` | 备份的旧 SQLite 实现 |
| `pages/3_数据浏览.py` | 6 个 tab（图书/名言/商品/新闻/招聘/通用）+ 关键字筛选 + SQL 透出 |
| `pages/4_统计仪表盘.py` | 6 个 Plotly 图表 + KPI 卡片，每个图配对应 SQL |
| `pages/5_SQL查询.py` | SQL 查询器，白名单安全过滤 + 11 条预设示例 + EXPLAIN |
| `pages/6_数据库结构.py` | 从 `information_schema` 动态展示表 / 列 / 索引 / 外键 / 约束 / 视图 / 触发器 |
| `.env.example` | 数据库连接配置模板 |

### 数据库设计

**元数据层（任务管理）：**
- `categories` — 类别字典（13 个）
- `jobs` — 采集任务（外键关联 categories）
- `results` — 每页抓取结果（外键关联 jobs，CASCADE 删除）

**维度表：**
- `authors` / `brands` / `news_sources` / `tags` — 范式化拆分

**业务专属表：**
- `books` / `quotes` / `products` / `news` / `jobs_listings`
- `generic_items` — 其他类别 JSON fallback

**关联表：**
- `quote_tags` — 名言-标签多对多

**审计表：**
- `price_history` — 价格变更历史，由触发器自动维护

**约束类型覆盖：**
- 主键 / 外键（CASCADE / SET NULL 双策略）
- UNIQUE（同书同作者只能一条；URL 哈希唯一）
- CHECK（price >= 0、rating BETWEEN 1 AND 5、birth_year 范围）
- 生成列（`news.url_hash` 用 SHA2(url, 256) 自动算）
- NOT NULL / DEFAULT

**索引覆盖：**
- 普通 B+树（scraped_at / price / rating）
- 复合索引（brand_id, price）
- 全文索引（books.title / quotes.quote / news.title+summary）

**视图：**
- `v_book_summary` / `v_top_authors` / `v_category_stats` / `v_product_summary`

**触发器：**
- `trg_book_price_change` / `trg_product_price_change` 自动写入 price_history

### 兼容性

- `database.py` 改为 shim，对原有 caller（`app.py` / `pipeline.py` / `pages/1_批量采集.py` / `pages/2_任务历史.py` / `scheduler.py`）零改动
- `pipeline.stage_store()` 新增 `category` 参数，按类别路由到专属表
- 智能模式分类完成后自动 `update_job_category()` 把 category 写回 jobs.category_id

### 旧数据迁移

- 8 个老任务 / 24 条 results / 240 行 quotes 数据全量迁移到 MySQL
- 类别推断启发式：按 `query` 字段命中（quote/sku/abstract 等专业字段优先）
- 失败行写入 `migration_failed.json` 供人工检查

### 依赖更新

`requirements.txt` 新增：
- `PyMySQL` — MySQL Python 驱动
- `SQLAlchemy>=2.0` — ORM 框架
- `plotly` — 仪表盘图表

### 一键导入数据库

```bash
mysql -u root -p --default-character-set=utf8mb4 < db/schema.sql
python -m db.migration   # 可选:迁移老 SQLite 数据
```
